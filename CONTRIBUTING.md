# Contribution Guidelines

## How to Contribute

We welcome contributions to the Phishing Detection System! Whether it's bug reports, feature requests, or code improvements, your help is appreciated.

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/phishing_project.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Commit: `git commit -m "Add descriptive message"`
6. Push: `git push origin feature/your-feature-name`
7. Open a Pull Request

### Code Style

- Follow PEP 8 guidelines for Python
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Include comments for complex logic

### Testing

Before submitting, ensure:
- Code runs without errors
- All imports are correct
- Output files are generated properly

### Issues

When reporting issues, please include:
- Python version and OS
- Steps to reproduce
- Error messages or logs
- Expected behavior

### Pull Requests

- Provide clear description of changes
- Reference any related issues
- Ensure code is clean and well-documented
- Keep commits organized and descriptive

Thank you for contributing!
